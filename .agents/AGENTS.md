# Project-Scoped Rules: Travel Agent

## Environment Constraints
- **CRITICAL**: Never overwrite, delete, or replace the local `.env` file (`c:\Projects\travel_agent\.env`).
- Always preserve any keys and configurations that the user has added to their local `.env` file.
- If you need to introduce new environment variables, ask the user to add them manually, or use code editing tools to insert/append them to `.env` without replacing the existing contents.
